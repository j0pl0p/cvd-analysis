# Репозиторий исследовательского проекта на тему «Анализ и визуализация данных с использованием Yandex DataLens: исследование на основе персональных индикаторов сердечно-сосудистых заболеваний»

## Структура репозитория

```
.
├── preprocessing/                          # Предобработка данных
│   ├── data/
│   │   ├── raw/                            # Исходные данные, опубликованные на Kaggle
│   │   └── processed/                      # Полученные данные после запуска пайплайна
│   ├── steps/                              # Вспомогательные скрипты для пошаговой предобработки
│   │   ├── binarizing.py                   # Бинаризация булевых данных к формату 0/1
│   │   ├── clearing_data.py                # Удаление данных (выбросы и т.д.)
│   │   ├── creating_target.py              # Создание целевой переменной
│   │   ├── merging.py                      # Объединение наборов данных
│   │   └── syncronizing.py                 # Подгон названий признаков и значений к одному формату
│   └── pipeline.py                         # Пайплайн предобработки
└── README.md
```

Из-за ограничений на размеры файлов в GitHub, в репозиторий нельзя добавить исходные и полученные данные, поэтому скачать и положить файлы

- `heart_2020_cleaned.csv`
- `heart_2022_with_nans.csv`

в папку `preprocessing/data/raw/` потребуется от читателя самостоятельно.

## Источники данных

В работе используются наборы данных, содержащие результаты опроса более 400 тыс.
взрослых, проводимого в рамках системы наблюдения за поведенческими
факторами риска (BRFSS). Опросы проводились Центрами по контролю и
профилактике заболеваний США (CDC) в 2020 и 2022 годах.

[Опубликованные наборы на Kaggle](https://www.kaggle.com/datasets/kamilpytlak/personal-key-indicators-of-heart-disease)

## Предобработка исходных данных

Запуск пайплайна предобработки в корне репозитория

```bash
python preprocessing/pipeline.py
```

### Вспомогательные скрипты

- `binarizing.py` \
    Переводит ответы респондентов формата `Yes`/`No` в значения `1`/`0` соответсвенно.

- `clearing_data.py`
  - Удаляет выбросы в данных, оставляя значения ИМТ (`BMI`) в диапазоне от 15 до 60 и сон (`SleepTime`) от 3 до 12 часов для набора 2020 года. Те же фильтры применяются для набора 2022 года, а также накладываются другие: сохраняются записи с ростом (`HeightInMeters`) в пределах от 1.4 до 2.2 метров и весом (`WeightInKilograms`) от 40 до 160 кг.
    - Удаляет противоречащие записи респондентов: беременных мужчин (параметр `Diabetic` равен `Yes (during pregnancy)`/`Yes, but only during pregnancy (female)`) для наборов 2020 и 2022 года и имеющие несоответсвующие ИМТ рост и вес для набора 2022 года.
    - Удаляет дубликаты.
- `creating_target.py` \
    Добавляет целевую переменную `HeartDisease` в набор 2022 года по формуле `HeartDisease = HadAngina OR HadHeartAttack`.
- `syncronizing.py` \
    Частично приводит наборы данных к единому формату:
  - переименовывает колонки в наборе 2020 года на аналогичные для 2022;
  - переименовывает значения в наборе 2022 года на аналогичные для 2020;
  - переводит признак `SmokerStatus` набора 2022 года в бинарный.
- `merging.py` \
    Объединяет наборы по диагонали, т.е. заполняя недостающие признаки пропущенными значениями.

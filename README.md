## Источники данных

В работе используются наборы данных, содержащие результаты опроса более 400 тыс.
взрослых, проводимого в рамках системы наблюдения за поведенческими
факторами риска (BRFSS). Опросы проводились Центрами по контролю и
профилактике заболеваний США (CDC) в 2020 и 2022 годах.

[Опубликованные наборы на Kaggle](https://www.kaggle.com/datasets/kamilpytlak/personal-key-indicators-of-heart-disease)

## Предобработка исходных данных

Запуск пайплайна предобработки в корне репозитория

```bash
python preprocessing/pipeline.py
```

### Вспомогательные скрипты

- `binarizing.py` \
    Переводит ответы респондентов формата `Yes`/`No` в значения `1`/`0` соответсвенно.

- `clearing_data.py`
  - Удаляет выбросы в данных, оставляя значения ИМТ (`BMI`) в диапазоне от 15 до 60 и сон (`SleepTime`) от 3 до 12 часов для набора 2020 года. Те же фильтры применяются для набора 2022 года, а также накладываются другие: сохраняются записи с ростом (`HeightInMeters`) в пределах от 1.4 до 2.2 метров и весом (`WeightInKilograms`) от 40 до 160 кг.
  - Удаляет противоречащие записи респондентов: беременных мужчин (параметр `Diabetic` равен `Yes (during pregnancy)`/`Yes, but only during pregnancy (female)`) для наборов 2020 и 2022 года и имеющие несоответсвующие ИМТ рост и вес для набора 2022 года.
- `creating_target.py` \
    Добавляет целевую переменную `HeartDisease` в набор 2022 года по формуле `HeartDisease = HadAngina OR HadHeartAttack`.
- `syncronizing.py` \
    Частично приводит наборы данных к единому формату:
  - переименовывает колонки в наборе 2020 года на аналогичные для 2022;
  - переименовывает значения в наборе 2022 года на аналогичные для 2020;
  - переводит признак `SmokerStatus` набора 2022 года в бинарный.
- `merging.py` \
    Объединяет наборы, сохраняя признаки 2020 года и добавляя колонку `Year`.

## Предварительные визуализации данных

Файлы, посвящённые визуализациям находятся в папке `visualizing/`. Скрипты находятся в папке `visualizing/scripts/` и запускаются по отдельности аналогично пайплайну предобработки:

```bash
python visualizing/scripts/age_group_percentage.py
```

Полученные визуализации появятся в папке `visualizing/plots/`.

*Важно: перед тем как запускать скрипты визуализаций, нужно прогнать предобработку данных, запустив пайплайн как указано выше.*

### Вспомогательные скрипты

- `age_group_percentage.py` \
   Сохраняет одноимённый график, изображающий процентное соотношение больных ССЗ участников за 2020 год по возрастным категориям.
- `boxplot_outliers.py` \
   Сохраняет усиковые диаграммы числовых столбцов наборов 2020 и 2022 года в файлы `boxplot_outliers_2020.png` и `boxplot_outliers_2022.png` соответсвенно.
- `null_count_2022.py` \
   Сохраняет одноимённый график, изображающий количество пропущенных значений каждого признака в наборе 2022 года.
- `target_imbalance.py` \
   Сохраняет одноимённый график, изображающий отношение значений целевой переменной `HeartDisease` в наборах данных 2020 и 2022 года.

## Работа с моделями машинного обучения

Файлы, посвящённые экспериментам с ML моделями находятся в папке `ml/`.

Эксперименты были проведены с моделями

- `CatBoostClassifier`
- `LogisticRegression`
- `RandomForestClassifier`

Соответсвующие jupyter-ноутбуки сохранены в папке `ml/models/`. Результаты проведенных экспериментов, такие как метрики моделей и `feature_importance` лучших из них сохранены в папке `ml/results/`.

Скрипт `results_merger.py` требуется запустить после исполнения всех трёх jupyter-ноутбуков, после чего он объединит результаты всех моделей в файлы

- `experiments.csv` \
  Метрики моделей на каждом наборе данных (ROC-AUC, F1-score, PR-AUC, Balanced Accuracy).
- `featute_importances.csv` \
  `feature_importance` лучших моделей на каждом наборе данных по метрике [Balanced Accuracy](https://evaluation-metrics-rewrite.vercel.app/docs/metrics/balanced-accuracy).

Краткая сводка экспериментов указана в таблице ниже

| Набор данных | Модель                 | Balanced Accuracy |
| ------------ | ---------------------- | ----------------: |
| 2020         | CatBoostClassifier     |            0.7664 |
| 2020         | LogisticRegression     |            0.7636 |
| 2020         | RandomForestClassifier |            0.7627 |
| 2022         | CatBoostClassifier     |            0.7765 |
| 2022         | LogisticRegression     |            0.7755 |
| 2022         | RandomForestClassifier |            0.7699 |
| merged       | CatBoostClassifier     |            0.7595 |
| merged       | RandomForestClassifier |            0.7575 |
| merged       | LogisticRegression     |            0.7548 |

Таким образом, `CatBoostClassifier` показал наилучшее качество на всех наборах данных.
